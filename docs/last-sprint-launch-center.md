# Sprint: centro de lanzamiento de Academies

Este sprint completa el flujo operativo previo a la comercialización. El panel de Administración ahora muestra una lista de lanzamiento que guía al equipo por los pasos necesarios para entregar una Academy:

1. Personalizar marca y bienvenida.
2. Elegir módulos.
3. Invitar al administrador del cliente.
4. Crear el primer perfil de facilitador.
5. Publicar el primer recurso.
6. Invitar al primer alumno.
7. Conectar y verificar el dominio personalizado.

El avance se calcula en backend junto con el contexto de la organización. Al completar el 100%, el panel muestra que la Academy está lista para piloto.

También se agregó el reenvío de invitaciones pendientes. Reenviar reutiliza la invitación existente, vuelve a enviar el acceso y actualiza su fecha; no crea duplicados.
